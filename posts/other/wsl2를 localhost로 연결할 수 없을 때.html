localhost 연결이 왜 안돼

wsl2(이하 wsl)에서

1python -m http.server

위와 같이 서버를 실행 한 다음, localhost:8000으로 접근하면 해당 디렉토리를 탐색할 수 있는 페이지에 접근 할 수 있다.

그런데 정확한 이유는 모르겠지만 이게 정상적으로 작동하지 않을 때가 있다.

wsl의 주소에 직접 접근하자

wslconfig를 수정해봐도 동일한 문제가 반복되는게 짜증나서, localhost대신 wsl의 로컬 주소를 매핑하는 글을 찾아봤다.

Fixing WSL2 localhost access issue

위 글에서 가져온, 아래에서 사용할 코드

1$hostname = "wslhost"
2
3# find ip of eth0
4$ifconfig = (wsl -- ip -4 addr show eth0)
5$ipPattern = "((\d+\.?){4})"
6$ip = ([regex]"inet $ipPattern").Match($ifconfig).Groups[1].Value
7if (-not $ip) {
8   exit
9}
10Write-Host $ip
11
12$hostsPath = "$env:windir/system32/drivers/etc/hosts"
13
14$hosts = (Get-Content -Path $hostsPath -Raw -ErrorAction Ignore)
15if ($null -eq $hosts) {
16   $hosts = ""
17}
18$hosts = $hosts.Trim()
19
20# update or add wsl ip
21$find = "$ipPattern\s+$hostname"
22$entry = "$ip $hostname"
23
24if ($hosts -match $find) {
25   $hosts = $hosts -replace $find, $entry
26}
27else {
28   $hosts = "$hosts`n$entry".Trim()
29}
30
31try {
32   $temp = "$hostsPath.new"
33   New-Item -Path $temp -ItemType File -Force | Out-Null
34   Set-Content -Path $temp $hosts
35
36   Move-Item -Path $temp -Destination $hostsPath -Force
37}
38catch {
39   Write-Error "cannot update wsl ip"
40}

해당 글에서는 wsl이 처음 시작하고, 네트워크가 연결되는 이벤트에 반응하는 스케쥴을 생성해서 관리하는 방식으로 되어있는데, 이전 글인 Distrod와 같이 쓰려니 알 수 없는 에러가 발생했다.

wsl으로 지속적인 서비스를 띄워놓을 생각은 없으므로, 터미널 접근시 실행하는 스크립트를 하나 작성하기로 했다.

1#!/usr/bin/env bash
2
3wsl_host_name=wslhost
4win_ps1='c:\\wsl\\remote-port-hosts.ps1'
5
6ip_pattern='([0-9]{1,3}\.){3}[0-9]{1,3}'
7win_hosts='/mnt/c/Windows/System32/drivers/etc/hosts'
8powershell_cmd='Start-Process powershell -verb runas -ArgumentList '\
9'"-ExecutionPolicy Bypass -File '${win_ps1}'"'
10
11# windows의 hosts에 작성되어 있는 wsl ip
12win_hosts_ip=`cat ${win_hosts} | \
13    grep ${wsl_host_name} | \
14    egrep -o ${ip_pattern}`
15# linux의 hosts에 작성되어 있는 wsl ip
16linux_hosts_ip=`cat /etc/hosts | \
17    grep ${wsl_host_name} | \
18    egrep -o ${ip_pattern}`
19# wsl이 실제로 실행되고 있는 ip
20real_ip=`ip addr | \
21    awk '/inet/ && /eth0/' | \
22    awk '{print $2}' | \
23    egrep -o ${ip_pattern}`
24
25if [ ${win_hosts_ip} == $real_ip ] && \
26    [ $linux_hosts_ip == $real_ip ]; then
27    exit 0
28fi
29
30# 관리자권한 파워쉘을 실행하여 hosts파일 수정
31echo run cmd to change windows hosts
32powershell.exe ${powershell_cmd}
33# 관리자권한으로 wsl의 hosts를 수정
34echo change /etc/hosts
35sudo sed -i s/${linux_hosts_ip}/${real_ip}/g /etc/hosts
36exit 0

이제 위 스크립트를 .zshrc와 같은 곳에서 실행하도록 설정하면 된다.

이 스크립트가 정상적으로 실행된다면, 이제 localhost대신 wslhost로 wsl에서 실행한 서비스에 접근할 수 있게 된다.

아쉽다

wsl내에서 윈도우의 hosts 파일을 수정할 수 있다면 훨씬 간단했지만, 쓰기 권한을 얻을 방법이 보이지 않아 일단 돌아서 처리하기로 했다.

좀 더 깔끔한 방법을 찾는다면 새로 포스팅하자.
