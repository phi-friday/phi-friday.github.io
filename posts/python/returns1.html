returns의 주요 기능들

returns는 파이썬 스크립트가 실행되면서 생성하는 값을 Container로 감싸고, 이 값을 개발자가 주도적으로 관리할 수 있도록 유도한다.

기본적으로 완성된 형태로 제공되는 Container는 Maybe, Result, IO, Future, RequiresContext가 있고, 다음과 같은 경우 사용된다.

  • Maybe: None 일 수 있는 값
  • Result: 에러가 발생 할 수 있는 값
  • IO: IO 작업과 관련된 값
  • Future: 비동기 작업과 관련된 값
  • RequiresContext: 함수의 의존성과 관련된 값

IOResult, RequiresContextFutureResult 등 결합된 형태로 제공하기도 한다. 물론 개발자가 직접 새로운 Container를 정의할 수도 있다.

이러한 Container들은 map, alt, bind, lash 메소드를 이용하여 최종 결과물로 안전하게 도달하고, 개발자는 그 Container의 상태를 확인하고, 그 값을 사용하면 된다.

Container의 내부 값에 영향을 미치는 map

설명보다는 코드를 직접 확인하는게 더 빠르다.

1from returns.maybe import Maybe, Nothing, Some
2from returns.result import Failure, ResultE, Success
3
4
5def div_3(value: float) -> float:
6    return value / 3
7
8
9dummy_exc = Exception("error")
10success_value: ResultE[int] = ResultE.from_value(1)
11maybe_value: Maybe[int] = Maybe.from_optional(1)
12failure_value: ResultE[int] = ResultE.from_failure(dummy_exc)
13maybe_non_value: Maybe[int] = Maybe.from_optional(None)
14
15assert success_value == Success(1)
16assert failure_value == Failure(dummy_exc)
17assert maybe_value == Some(1)
18assert maybe_non_value == Nothing
19
20success_result = success_value.map(div_3)
21failure_result = failure_value.map(div_3)
22maybe_result = maybe_value.map(div_3)
23maybe_non_result = maybe_non_value.map(div_3)
24
25assert success_result == Success(div_3(1))
26assert failure_result is failure_value
27assert maybe_result == Some(div_3(1))
28assert maybe_non_result is Nothing
29
30assert success_result.unwrap() == div_3(1)
31assert failure_result.failure() is dummy_exc
32assert maybe_result.value_or(None) == div_3(1)
33assert maybe_non_result.value_or("test") == "test"

Container의 내부 값을 사용하는 함수를 map메소드의 인수로 전달하면, 그 Container의 상태가 정상일 때만 작동하고, 그렇지 않다면 아무 일도 하지 않는다.

즉, map은 정상적인 상태인 Container의 값만 변화시키는 메소드다.

map에 사용되는 함수는 에러가 발생하지 않는 순수함수를 가정하고 사용되기에, 함수 실행중 에러가 발생하더라도 Result 객체로 감싸지지 않고 작동을 멈출 것이다.

그렇다면 비정상 상태의 Container 내부의 값을 변화시키려면 어떻게 해야할까? 그때는 alt 메소드를 사용하면 된다.

Container의 상태를 변화시키는 bind

1from returns.maybe import Maybe, Nothing, Some, maybe
2from returns.result import Failure, ResultE, Success, safe
3
4
5def div_3_v1(value: int) -> ResultE[float]:
6    try:
7        return Success(3 / value)
8    except Exception as exc:
9        return Failure(exc)
10
11
12@safe
13def div_3_v2(value: int) -> float:
14    return 3 / value
15
16
17def div_3_v3(value: int) -> Maybe[float]:
18    try:
19        return Some(3 / value)
20    except:
21        return Nothing
22
23
24@maybe
25def div_3_v4(value: int) -> float | None:
26    return 3 / value
27
28
29dummy_exc = Exception("error")
30success_value: ResultE[int] = ResultE.from_value(0)
31maybe_value: Maybe[int] = Maybe.from_optional(1)
32failure_value: ResultE[int] = ResultE.from_failure(dummy_exc)
33maybe_non_value: Maybe[int] = Maybe.from_optional(None)
34
35assert success_value == Success(0)
36assert failure_value == Failure(dummy_exc)
37assert maybe_value == Some(1)
38assert maybe_non_value == Nothing
39
40success_result = success_value.bind(div_3_v1)
41failure_result = failure_value.bind(div_3_v2)
42maybe_result = maybe_value.bind(div_3_v3)
43maybe_non_result = maybe_non_value.bind(div_3_v4)
44
45assert str(success_result) == "<Failure: division by zero>"
46assert failure_result is failure_value
47assert maybe_result == div_3_v4(1)
48assert maybe_non_result is Nothing
49
50assert repr(success_result.failure()) == "ZeroDivisionError('division by zero')"
51assert failure_result.failure() is dummy_exc
52assert maybe_result.value_or(None) == 3
53assert maybe_non_result.value_or("test") == "test"

bind는 Container의 값을 사용하여 새로운 Container를 반환하는 함수를 받는다. 그 함수가 어떻게 정의됐는가에 따라 정상 상태의 Container를 반환할 수도 있고, 비정상 상태의 Container를 반환할 수도 있다.

즉, bind는 정상적인 상태인 Container의 값과 상태를 변화시키는 메소드다.

이러한 함수를 div_3_v1이나 div_3_v3처럼 일일이 정의하는 것은 손이 많이 가는 작업이므로, 가능하다면 div_3_v2이나 div_3_v4처럼 safe나 maybe와 같은 데코레이터를 사용할 것을 권장한다.

map과 alt처럼 bind에도 대응하는 메소드가 있는데, 바로 lash다. lash는 비정상 상태의 컨테이너를 정상 상태의 컨테이너로 바꿀 수 있다. 물론 비정상 상태를 유지할 수도 있다.

여러개의 Container를 함께 사용하는 방법

Container 각각에 대해 map, bind 등을 사용하여 값과 상태를 관리하는 것은 알겠지만, 두개 이상의 Container를 복합적으로 사용하려면 어떻게 해야할까?

여러 방법이 있겠지만, 그 중 세가지만 알아보자.

returns는 mypy를 사용한다고 가정하기에, pylance에서는 정상적으로 타입추론이 되지 않는 경우가 있다.

대부분의 경우 문제가 없었는데, 후술할 curry가 그러했다.

개별적으로 사용하는 방법

1from returns.curry import curry, partial
2from returns.io import IO
3
4
5def sum_v1(left: float, right: float) -> float:
6    return left + right
7
8
9sum_v2 = curry(sum_v1)
10
11left: IO[float] = IO(1.0)
12right: IO[float] = IO(2.0)
13
14assert right.apply(left.apply(IO(sum_v2))) == IO(3.0) # pylance에서 문제 발생
15assert right.apply(left.apply(IO(lambda x: partial(sum_v1, x)))) == IO(3.0)
16assert right.apply(left.apply(IO(lambda x: lambda y: sum_v1(x, y)))) == IO(3.0)
17assert sum_v2(1.0)(2.0) == 3 # pylance에서 문제 발생

returns에서 제공하는 curry를 사용하면, 여러개의 인자를 필요로 하는 함수에 인자를 전달할 때, 한번에 모두 전달하지 않아도 된다. 물론 lambda와 partial을 사용해서 직접 구현해도 문제 없다.

여기서 사용되는 partial은 functools와 같은 방식으로 사용된다. 차이가 있다면 returns의 partial은 좀 더 타입추론을 똑똑하게 해준다.

성능은 lambda로 직접 구현하는게 더 낫다고 한다..

Iterable로 묶어서 사용하는 방법

1from typing import Callable
2
3from returns.curry import partial
4from returns.io import IO
5from returns.iterables import Fold
6
7
8def sum_v1(left: float, right: float) -> float:
9    return left + right
10
11
12def sum_v2(value: float) -> Callable[[float], float]:
13    return partial(sum_v1, value)
14
15
16def get_one() -> IO[float]:
17    return IO(1)
18
19
20assert Fold.loop([get_one() for _ in range(10)], IO(5), sum_v2) == IO(10 + 5)
21assert Fold.loop([get_one() for _ in range(5)], IO(7), sum_v2) == IO(5 + 7)

Fold.loop는 Iterable로 묶여진 Container를 사용하는데 유용한 클래스메소드다. 내부 알고리즘은 단순하며, 다음과 같다.

  1. Iterable[Container[T]]에서 Container[T] 하나를 선택한다.

  2. 선택한 Container[T]로 Callable[[R], R] 객체를 생성한다. (T가 아니다!)

  3. 초기값으로 지정된 Container[R]를 사용하여 새로운 초기값 Container[R]을 생성한다.

  4. 반복

여러개로 나눠진 Container를 하나로 합치는 방법

1from returns.converters import maybe_to_result
2from returns.iterables import Fold
3from returns.maybe import Maybe, Nothing, Some, maybe
4from returns.result import Failure, Result, Success
5
6init_data = dict(zip(list("abc"), range(3)))
7
8
9@maybe
10def get(key: str) -> int | None:
11    return init_data.get(key)
12
13
14values_abc: list[Maybe[int]] = [get(key) for key in "abc"]
15values_abcd: list[Maybe[int]] = [get(key) for key in "abcd"]
16result_abc: list[Result[int, None]] = [maybe_to_result(x) for x in values_abc]
17result_abcd: list[Result[int, None]] = [maybe_to_result(x) for x in values_abcd]
18
19assert Fold.collect(values_abc, Some(())) == Some((0, 1, 2))
20assert Fold.collect(result_abc, Success(())) == Success((0, 1, 2))
21assert Fold.collect(values_abc, Some((100,))) == Some((100, 0, 1, 2))
22assert Fold.collect(result_abc, Success((200,))) == Success((200, 0, 1, 2))
23
24assert Fold.collect(values_abcd, Some(())) is Nothing
25assert Fold.collect(result_abcd, Success(())) == Failure(None)
26
27assert Fold.collect_all(values_abcd, Some(())) == Some((0, 1, 2))
28assert Fold.collect_all(result_abcd, Success(())) == Success((0, 1, 2))
29assert Fold.collect_all(values_abcd, Some((10, 20))) == Some((10, 20, 0, 1, 2))
30assert Fold.collect_all(result_abcd, Success((10, 20))) == Success((10, 20, 0, 1, 2))

Fold.collect를 사용하면, 모든 Container가 정상인 경우에 대해서만 수집을 진행하고,

Fold.collect_all를 사용하면, 비정상 상태의 Container를 제외하고 수집을 진행한다.
